How does JAVASCRIPT work.

javascript is interpreted, compiled, single-theted,event-givent language. It's a client-side scripting language.Its run both of browser and server . When we write code in our code editor, its reads line by line and send the code to the browser engine. The browser engine converts into machine language and shows the output. by default, all data is stored in the global object and the default value = undefined. When execution will start all data values = Are defined (means show).When we create a function, all the function data are stored in the browser memory. When we call the function it will create a different execution context. Here are two types of scope first one global scope and the second one is block scope. We can access global variables anywhere in our code. But we can't access block variables anywhere in our code cause its blocked.

How does EVENT LOOP work.

Event loop: 1. render Queue 2. call back queue 3. microtask queue.
When we set synchronous function it will reach call stuck in the browser. But when we set an asynchronous function it will reach web api. If my call stuck is busy, my asynchronous function is processing web api. When my call stuck is free my processing asynchronous function reaches call back queue and call back queue sent the processing asynchronous function to call stack and finally showing the output in the browser. The name of the combination between the two functions (call back queue and call stack)is EVENT LOOP. When we fetch big data from anywhere and show it in our UI if we do it synchronously . The browser hangs until our data is loaded.When all data is loaded then the event loop call the rander queue and show the output in the browser. It's a lengthy process.In this situation event loop help to fetch our data easily. There is no fear of crashing our program.When we create a promise(new promise) in our function it will store browser api. When my call stuck is is free browser api sent my (promise function) to (microtask queue).Event loop gives it first priority and sent call stuck before call back queue initialize.
